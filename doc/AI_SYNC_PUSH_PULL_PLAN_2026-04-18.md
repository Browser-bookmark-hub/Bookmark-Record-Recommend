## 0.1 当前代码落地状态（v0 骨架）

已在当前项目落地（2026-04-18）：
1. 左侧新增第 4 个一级栏目：`同步/AI分析`。
2. 同步页支持手动按钮：`拉取覆盖` / `推送当前` / `同步（推送+拉取）`。
3. 同步页支持“推送数据包勾选（6 项）”：推荐数据池、推荐行为事件、书签记录综合包、时间捕捉包、书签树快照包、高体量原始包（默认关闭）。
4. 同步页支持“左侧文件列表 + 右侧单 MD 渲染器 + 可选编辑/保存”。
5. 拉取覆盖策略支持开关：`拉取时保留本地已编辑文件`。

说明：
- 当前实现同时支持两种模式：`GitHub Repo`（主模式）与本地 `storage.local` 快照（调试回退）。
- GitHub 模式已提供 Repo 配置输入（Token/Owner/Repo/Branch/BasePath）与连接测试按钮。
- Push 会写入 `data/latest.json`、`data/snapshots/...`、`meta/sync_state.json`；Pull 会读取 `ai/results` 下的 Markdown 结果。

## 0. 一句话结论

本项目先做 **GitHub Repo 手动同步**：
- `Push` 上传分析数据（机器读）
- `Pull` 只拉 AI 结果（主要是 `MD`）
- 结果在扩展内新增“同步”页做 **单一 MD 渲染展示**

当前不做：
- 轮询自动同步
- Google Drive / OneDrive Provider

---

## 1. 已确认决策（本轮冻结）

1. 同步方式：仅手动点击（`Push` / `Pull` / `Push+Pull`）。
2. 模式：`Push` 与 `Pull` 分离。
3. 拉取策略：只拉 AI 结果路径，不反拉整套原始数据。
4. 结果展示：先做只读 MD 渲染。
5. Provider：先只做 GitHub Repo。
6. 本阶段上传格式：只上传 JSON/JSONL 数据包，不上传 TSV。
7. 云端人工查看（表格化视图）本阶段暂不做，后续接 Google Drive / OneDrive 时再加 TSV 方案。

---

## 2. 格式分工（避免混乱）

1. **JSON / JSONL**：主数据协议（机器分析真源，当前唯一上传数据格式）。
2. **MD**：AI 结果与说明文档（前端渲染展示）。
3. **TSV**：延期到多云盘阶段（Google Drive / OneDrive）再引入。

结论：本阶段固定为“**数据用 JSON，结果用 MD**”。

## 4. AGENTS.md 规则文件（拟定）

新增文件：`bookmark_record_and_recommend_sync/AGENTS.md`，用途：

1. 规定 AI 允许读取的输入路径。
2. 规定 AI 必须输出的路径与格式。
3. 规定输出模板（标题、摘要、候选建议、风险说明）。
4. 规定覆盖策略（始终覆盖 `latest.md`，可选追加 `runs`）。

建议输出最小模板：

```md
# AI Recommend Report
## Summary
## Top Candidates
## Review Later Suggestions
## Block / Skip Suggestions
## Why (Signals)
## Next Sync Hints
```

## 5. MD 渲染器结论

### 5.1 是否需要多个 MD 渲染器？

不需要。先做一个统一渲染器即可。

### 5.2 渲染策略（建议）

1. 仅支持安全 Markdown 子集（标题、段落、列表、表格、代码块、链接）。
2. 默认禁用内联 HTML 脚本能力（防注入）。
3. 外链点击走新标签打开。
4. 图片/嵌入先关闭，后续按需放开。

## 6. 本项目“推送哪些数据”已确认清单（v1）

你已确认的方向：重点推“推荐池 + 行为 + 记录 + 时间 + 树快照”，而不是只推零散字段。

## 6.1 必推（默认勾选）：推荐数据池（S 数据池）包

用途：把推荐系统当前状态完整交给 AI，支持分析、复算、解释。
包含内容（功能语义）：
1. 候选池全量快照（不是只推当前三卡）。
2. 每条候选当前 `S` 与分项信号（`F/C/T/D/L/R`）。
3. 当前排序位次、游标、模式参数与版本信息。
4. 屏蔽/待复习/跳过/复习等推荐相关状态。

## 6.2 必推（默认勾选）：推荐行为事件包

用途：让 AI 看到用户如何“处理推荐”，而不是只看静态分值。
包含内容（功能语义）：
1. 打开、跳过、待复习、屏蔽、复习等行为事件。
2. 事件时间、对象标识、来源场景（推荐页/小组件等）。
3. 事件按时间分区（便于日/周/月回放分析）。

## 6.3 必推（默认勾选）：书签记录综合包

用途：提供“书签记录”模块的核心分析底座。
包含内容（功能语义）：
1. 书签添加记录（按时间组织）。
2. 点击记录（按时间组织）。
3. 点击排行数据。
4. 关联记录数据。

说明：你提到的“书签记录本身已有时间区分和统装”沿用现有结构，不重复拆分。

## 6.4 必推（默认勾选）：时间捕捉包

用途：补充阅读深度与活跃强度信号。
包含内容（功能语义）：
1. 时间捕捉聚合（天/周/月/年）。
2. 关键条目时长与唤醒统计。
3. 与推荐分析相关的摘要指标。

## 6.5 必推（默认勾选）：书签树快照包

用途：提供目录上下文，便于 AI 解释“为什么推荐”。
包含内容（功能语义）：
1. 书签树层级结构快照。
2. 文件夹与书签基础元信息。
3. 快照版本与生成时间。

## 6.6 可选推（默认关闭）：高体量原始包

用途：排障与深度调优（普通分析默认不需要）。
包含内容（功能语义）：
1. 原始浏览/行为 JSONL 明细。
2. 更细粒度时间会话明细。
3. 推荐诊断与调试元数据。

## 6.7 暂不推（本阶段禁推）

1. 与推荐分析无关的纯 UI 偏好缓存。
2. 图标缓存、临时过渡缓存等噪声数据。
3. 未经脱敏的隐私高敏字段。
