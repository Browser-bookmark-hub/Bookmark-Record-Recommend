# 小组件图表视图切换计划（轻量版，2026-04-08）

## 目标（本轮）
1. 不删除原有小组件展示，只新增“视图切换”。
2. `书签添加记录` 与 `点击记录` 新增 X/Y 曲线图（支持悬停显示日期与数量）。
3. `点击排行` 新增饼图视图，但默认仍是原列表视图。
4. 兼容侧边栏宽度变化（展开/收起）与标签页宽度变化，避免布局抖动。
5. 延续“先视觉后补算”策略，避免首开卡顿与滚动掉帧。

## 默认行为（确认版）
1. `书签添加记录` 默认视图为 `曲线图`。
2. `点击记录` 默认视图为 `曲线图`。
3. 上述两卡默认范围为 `本周`，可切换到 `本月(近30天)`。
4. `点击排行` 默认视图为 `列表`，可切换到 `饼图`。
5. 原有周数字格/列表均保留，作为可切换视图与兜底。

## 交互设计
1. 每张卡片头部新增 `视图切换按钮`（不替换现有按钮，按当前布局并列）。
2. `书签添加记录` / `点击记录` 额外新增 `范围切换按钮`：`本周 <-> 本月`。
3. `点击排行` 新增 `列表 <-> 饼图` 切换按钮；现有 `模式`、`范围` 按钮继续生效。
4. 曲线图与饼图均支持 hover 提示：
   - 曲线图：`X=日期`，`Y=数量`
   - 饼图：`项名称 + 数值 + 占比`

## 持久化（localStorage）
1. `widgetsAdditionsViewMode`: `line | week-grid`
2. `widgetsAdditionsRange`: `week | month30`
3. `widgetsHistoryViewMode`: `line | week-grid`
4. `widgetsHistoryRange`: `week | month30`
5. `widgetsRankingVisualMode`: `list | pie`

说明：
- 只新增 UI 视图偏好键，不改已有统计数据键。
- 现有 `browsingRankingViewMode`、`timeTrackingWidgetRankingRange` 继续沿用。

## 图表技术路线（轻量版）
1. 使用原生 `SVG + CSS + 原生事件` 实现折线图/饼图，不引入外部图表库。
2. 图表渲染和现有小组件刷新共用同一套刷新节奏（脏标记、首屏保护、空闲补算）。
3. 保留原有周格/列表视图作为稳定兜底，随时可切回。
4. 视图切换只更新当前卡片，不触发整页重算。

备注：
- 轻量版优先降低首开负担和维护复杂度；后续如需更复杂交互，再评估引入图库。

## 数据口径与计算
1. `书签添加记录-本周`：继续复用现有 `dateAdded` 周统计。
2. `书签添加记录-本月(30天)`：基于 `dateAdded` 生成近30天日序列。
3. `点击记录-本周`：继续复用现有浏览历史周统计缓存。
4. `点击记录-本月(30天)`：复用现有历史缓存分段读取（近30天）。
5. `点击排行-饼图`：复用当前排行结果（同模式、同范围），展示 TopN + 其他。

## 性能与调度（必须执行）
1. 首次进入 widgets：
   - 先渲染 loading/已有缓存；
   - 图表重计算不穿透视觉保护窗口。
2. 非默认范围（本月）默认“点到再算”，不参与首屏强算。
3. 交互活跃期（滚动/拖拽/连续点击）只标脏，不立刻重算。
4. 空闲时补算本月数据与图表缓存，避免下一次切换等待。
5. 图表更新需校验当前视图 token，旧请求晚到直接丢弃。

## 响应式与布局
1. 监听卡片容器宽度变化（`ResizeObserver` + 节流）更新图表尺寸。
2. 侧边栏 `compact/hidden/collapsed` 时：
   - 停止图表动画；
   - 必要时销毁实例，恢复展开后重建。
3. 侧边栏展开过渡期间延后图表重绘，避免“横向拉伸闪烁”。

## 动效策略
1. 首次绘制使用短时入场动画（约 300~500ms）。
2. 数据切换动画保守（约 220~320ms），避免频繁重绘导致掉帧。
3. 低性能场景自动降级：关闭阴影与复杂过渡，仅保留基础动画。

## 实施批次
1. Phase A：新增按钮与持久化骨架（不接图表，仅切换占位）。
2. Phase B：接入折线图（添加记录/点击记录）+ 本周/本月口径。
3. Phase C：接入点击排行饼图 + TopN 合并“其他”。
4. Phase D：侧边栏宽度变化联动、交互保护与兜底收敛。

## 当前落地状态（本轮）

## 验收标准
1. 默认打开时：
   - 添加记录、点击记录显示曲线图；
   - 点击排行显示列表。
2. 切换按钮后关闭再打开，视图与范围保持上次选择。
3. 滚动 widgets 区域不出现明显掉帧放大。
4. 图表失败时不出现空白，原有周格/列表可正常回退。
5. 点击图表点位/扇区能显示明确 X/Y（或名称/占比）信息。

## 与现有计划的关系
1. 本计划不修改推荐 `S` 算法。
2. 本计划不改“后台统一口径”数据结构。
3. 本计划遵守《小组件严格版先视觉后补算计划（2026-04-08）》调度规则。
