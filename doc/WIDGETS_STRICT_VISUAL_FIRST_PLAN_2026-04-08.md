# 小组件严格版「先视觉后补算」计划（2026-04-08）

## 目标
1. 首屏只计算用户默认可见口径，优先展示已有缓存/共享状态。
2. 非默认口径统一改为「用户点到再算」或「空闲补算」。
3. 交互过程中（滚动/切换/拖拽）不执行重补刷，避免掉帧。

## 严格版口径规则
### 首屏默认口径（白名单）
1. 推荐卡：当前轮次（`historyCurrentCards` / snapshot）。
2. 点击排行：默认 `day + bookmark`。
3. 书签添加记录：默认本周。
4. 点击记录：默认本周。
5. 关联记录：默认 `day + level1`。

### 非默认口径
1. `ranking` 的 `week/month/year/all` 与 `folder/domain/subdomain`。
2. `related` 的 `week/month/year/all`、`level2`、二级筛选。
3. 非默认口径默认不参与首屏计算。

## 调度规则
1. 优先级：`首屏白名单 > 用户点到再算 > 空闲补算`。
2. 交互活跃时只排队不执行，交互结束后再补刷。
3. 结果提交必须匹配当前视图/当前口径，旧结果晚到直接丢弃。

## 第一批落地（本轮执行）
1. 收窄 widgets 首开刷新风暴：首开阶段减少高频重算目标。
2. 引入首开「视觉窗口」：视觉窗口内禁止重路径统计链路抢主线程。
3. `ranking/related` 在视觉窗口优先走快路径/缓存，重路径延后。
4. 周期刷新与补刷默认不允许穿透交互保护。
5. 用户主动切换口径（按钮点击）保留「点到即算」能力。
